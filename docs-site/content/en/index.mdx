# nxspub Documentation

If you are new to release tooling, this documentation is written so you can go from zero to a working pipeline.

`nxspub` helps you do 4 things:

- compute versions from commit history
- generate changelogs
- publish packages (`release`)
- deploy and rollback (`deploy`)

## Shortest Working Release Path

```bash
nxspub version
nxspub release
```

The two commands above are enough to complete version bump + package publish.

If you are not deploying yet, you can skip `deploy` for now.
