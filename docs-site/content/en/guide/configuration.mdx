# Configuration

Create `nxspub.config.ts` in your repository root. If you are new, start with minimal config, then add advanced blocks.

## Minimal Config for Beginners

```ts
import type { NxspubConfig } from 'nxspub'

const config: NxspubConfig = {
  branches: {
    main: 'latest',
  },
}

export default config
```

With only this config, you can already run:

```bash
nxspub version
nxspub release
```

## Full Example for Teams

```ts
import type { NxspubConfig } from 'nxspub'

const config: NxspubConfig = {
  workspace: {
    mode: 'independent',
    passive: 'follow',
  },
  branches: {
    main: 'latest',
    alpha: 'preminor',
    beta: 'preminor',
    hotfix: 'patch',
  },
  versioning: {
    major: [/BREAKING CHANGE:/],
    minor: [/^feat(\(.+\))?:/],
    patch: [/^fix(\(.+\))?:/, /^perf(\(.+\))?:/],
  },
  changelog: {
    labels: {
      feat: 'Features',
      fix: 'Bug Fixes',
      perf: 'Performance',
      refactor: 'Refactors',
    },
    writeOnBranches: ['main', 'master'],
  },
  lint: {
    'commit-msg': {
      pattern:
        /^(revert: )?(feat|fix|docs|dx|style|refactor|perf|test|workflow|build|ci|chore|types|wip|release)(\([^)]+\))?(!)?: .{1,50}/,
      message: 'Invalid commit message format.',
    },
  },
  'git-hooks': {
    'pre-commit': 'pnpm lint-staged && pnpm check',
    'commit-msg': 'npx nxspub lint --edit "$1"',
  },
  scripts: {
    releaseBuild: 'pnpm -r build',
  },
  deploy: {
    enabled: true,
    defaultEnvironment: 'staging',
    branchEnvironmentMap: {
      main: 'production',
      alpha: 'staging',
      beta: 'staging',
    },
    provider: {
      name: 'custom',
      config: {
        requiredEnv: ['DEPLOY_TOKEN'],
        commands: {
          validate: 'echo validate',
          plan: 'echo plan',
          execute: 'echo deploy',
          rollback: 'echo rollback',
        },
      },
    },
    environments: {
      staging: { strategy: 'rolling' },
      production: { strategy: 'canary' },
    },
    promotion: {
      requireSameArtifactDigest: true,
      sourceEnvironment: 'staging',
    },
  },
}

export default config
```

## 4 Deploy Fields Beginners Usually Find Confusing

- `environments.staging.strategy` / `environments.production.strategy`
  - this is the rollout method per environment.
  - beginner-friendly default: `staging = rolling`, `production = canary`.
- `requiredEnv`
  - list of required environment variables.
  - example: `['DEPLOY_TOKEN']` means deploy is denied if token is missing.
- `commands.plan / execute / rollback`
  - three phases: preview / deploy / rollback.
  - start with `echo` placeholders, replace with real scripts after flow is verified.
- `--strategy` CLI flag
  - temporary override for one run, for example:
  - `nxspub deploy --env production --strategy canary`
  - this takes precedence over default strategy in config for that command execution.

## 3 Most Important Config Blocks

- `branches`: controls allowed versioning policy per branch.
- `changelog.writeOnBranches`: controls where `CHANGELOG.md` can be written directly.
- `lint['commit-msg']`: prevents bad commit messages from triggering wrong bumps.

## Branch Policy Tips (Practical)

- Use `latest` only for stable branches (`main`/`master`).
- Use `pre*` policies for prerelease branches (`alpha`, `beta`).
- Use `patch` for hotfix branch if you want to block `feat`/`breaking` commits.

## Changelog Write Strategy

- `changelog.writeOnBranches` limits where `CHANGELOG.md` is written.
- On non-allowlisted branches, nxspub stores draft files under `.nxspub/changelog-drafts/*`.
- Stable branch run can import those drafts automatically.
