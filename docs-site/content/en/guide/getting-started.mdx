# Getting Started

Goal of this page: run your first `version + release` flow in about 10 minutes.

## 1. Prerequisites

- Node.js `>=20`
- Git repository
- `pnpm` recommended (npm/yarn also supported)

## 2. Install

<PackageManagerTabs>
  <PackageManagerTabs.Tab>

```bash
pnpm add -D nxspub
```

  </PackageManagerTabs.Tab>


## 3. Create Config File

Create `nxspub.config.ts`:

```ts
import type { NxspubConfig } from 'nxspub'

const config: NxspubConfig = {
  branches: {
    main: 'latest',
    alpha: 'preminor',
    beta: 'preminor',
    hotfix: 'patch',
  },
}

export default config
```

## 4. Run Safe Preview First (No File Changes)

```bash
nxspub version --dry
```

You should see target version and changelog preview. If they look right, continue.

## 5. Run Real Versioning

```bash
nxspub version
```

This updates:

- `package.json` version
- `CHANGELOG.md`
- git commit and tag

## 6. Publish (Dry Run First)

```bash
nxspub release --dry
nxspub release
```

## 7. If Something Fails

- If branch is denied, check `branches` policy in config.
- If tag already exists, verify release state and choose next version properly.
- If you only want to inspect behavior, use `--dry`.
