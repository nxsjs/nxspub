# Quick Start Workflows

This page covers 3 practical flows you can run directly.

## Scenario A: Single Package Stable Release (main)

1. Configure branch policy:
```ts
branches: { main: 'latest' }
```
2. Use a valid commit message:
```bash
feat: add xxx
fix: correct yyy
```
3. Run version flow:
```bash
nxspub version
```
4. Publish:
```bash
nxspub release
```

## Scenario B: Prerelease Flow (alpha)

1. Configure branch policy:
```ts
branches: {
  main: 'latest',
  alpha: 'preminor',
}
```
2. On `alpha`, run:
```bash
nxspub version
nxspub release --tag alpha
```
3. Merge to `main`, then run stable release:
```bash
nxspub version
nxspub release
```

## Scenario C: Workspace Multi-Package Release

1. Configure workspace mode:
```ts
workspace: { mode: 'independent', passive: 'follow' }
```
2. Run version in repo root:
```bash
nxspub version
```
3. Run release in repo root:
```bash
nxspub release
```

## Common Safety Commands

```bash
nxspub console
nxspub console --web --open
nxspub version --dry
nxspub release --dry
nxspub deploy --plan
```
