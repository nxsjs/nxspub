# Commands

All commands support `--cwd <path>`.  
If this is your first time, read in this order: `version -> release -> deploy -> console`.

## version (Start Here)

<PackageManagerTabs>
  <PackageManagerTabs.Tab>

```bash
pnpm dlx nxspub version
pnpm dlx nxspub version --dry
```

  </PackageManagerTabs.Tab>
  <PackageManagerTabs.Tab>

```bash
npx nxspub version
npx nxspub version --dry
```

  </PackageManagerTabs.Tab>
  <PackageManagerTabs.Tab>

```bash
yarn dlx nxspub version
yarn dlx nxspub version --dry
```

  </PackageManagerTabs.Tab>
</PackageManagerTabs>

What it does:

- computes next version
- updates changelog
- creates git commit and tag


  </PackageManagerTabs.Tab>
  <PackageManagerTabs.Tab>

```bash

```

  </PackageManagerTabs.Tab>
</PackageManagerTabs>

What it does:

- publishes the version currently in `package.json`
- usually run `--dry` first
- prerelease branches commonly use `--tag alpha` or `--tag beta`

## deploy (Ship to Runtime Environment)

<PackageManagerTabs>
  <PackageManagerTabs.Tab>

```bash
pnpm dlx nxspub deploy --plan
pnpm dlx nxspub deploy --env production --strategy canary
pnpm dlx nxspub deploy --rollback --to <deploymentId>
```

  </PackageManagerTabs.Tab>
  <PackageManagerTabs.Tab>

```bash
npx nxspub deploy --plan
npx nxspub deploy --env production --strategy canary
npx nxspub deploy --rollback --to <deploymentId>
```

  </PackageManagerTabs.Tab>
  <PackageManagerTabs.Tab>

```bash
yarn dlx nxspub deploy --plan
yarn dlx nxspub deploy --env production --strategy canary
yarn dlx nxspub deploy --rollback --to <deploymentId>
```

  </PackageManagerTabs.Tab>
</PackageManagerTabs>

What it does:

- `--plan` only computes and validates plan
- runtime deploy requires `deploy.provider` config
- rollback requires `--rollback --to <deploymentId>`

  </PackageManagerTabs.Tab>

What it does:

- web mode: `nxspub console --web --open`
- strict read-only auditing: `--readonly-strict`
- gives teams a clear view of what will happen before applying changes

## draft-doctor (Draft Repair Tool)

```bash
nxspub draft-doctor
nxspub draft-doctor --target 1.3.0
nxspub draft-doctor --target 1.3.0 --prune
```

## git-hooks / lint (Commit-Time Validation)

```bash
nxspub git-hooks
nxspub git-hooks --dry
nxspub lint --edit .git/COMMIT_EDITMSG
```
